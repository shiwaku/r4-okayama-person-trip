# 岡山県パーソントリップ調査マスターデータ

## 概要

令和4年（2022年）実施の岡山県パーソントリップ調査のマスターデータ。15歳以上の岡山県民を対象とした移動実態調査。

- 対象人口: 1,571,431人（R4.10月時点の15歳以上）
- 調査日: 平日 10/12(水)・10/19(水)、休日 10/16(日)・10/23(日)
- 回収票数: 平日 27,395票（標本率1.74%）、休日 27,228票（標本率1.73%）

### ゾーンコード表シート（2,799行×21列）

全国の住所コード（10桁）とR03Bゾーンコードの対応表。

- 住所コード（10桁）= 都道府県+市区町村（5桁） + 大字・町（3桁） + 丁目（2桁）
- R03Bゾーンコード: 2桁+3桁+2桁の構成
- 集約方針: 岡山県はそのまま市区町村単位、兵庫・鳥取・広島は市町村単位、その他は県単位

### ターミナルコード表シート（7,517行×19列）

バス停・駅・電停・港・空港のコード（4桁）と名称の対応表。全国網羅。

## 拡大係数の設定（04_拡大係数の設定.pdf）

### 算出方法

```
拡大係数（仮値）c = 人口（b） / サンプル数（a）
```

平休別・性別・年齢階層別・市区町村別に設定。サンプル数が0または極端に少ない場合は近隣階層と統合。

### 対象地域（市区町村単位）

岡山市（北区・中区・東区・南区）、倉敷市、津山市、玉野市、笠岡市、井原市、総社市、高梁市、新見市、備前市、瀬戸内市、赤磐市、和気町、早島町、里庄町、矢掛町、新庄村、鏡野町、勝央町、奈義町、西粟倉村、久米南町 ほか

### 精度検証結果

| 区分 | 移動回数 | 相対誤差 | 精度確保ゾーン数 |
|---|---|---|---|
| 平日 | 2.23トリップ/日 | 19.56% | 24ゾーン（1目的2手段） |
| 休日 | 1.78トリップ/日 | 19.30% | 21ゾーン（1目的2手段） |

相対誤差の計算式: `RSD(A) = K × √((zk-1)/(1-r)/r/N)`（K=1.96、N=母集団、r=標本率、zk=カテゴリー数）

集計分析は移動詳細が把握可能な**6回まで**の移動を対象とする（7回以上は移動回数の加算のみ）。

## 生成済みファイル

### スクリプト（scripts/）

| ファイル | 内容 |
|---|---|
| `make_trips_full.py` | 平日・休日マスターデータの全94列にゾーン座標を結合し、コード値を日本語ラベルに変換して `trips_full.csv` を出力 |
| `make_od_lines.py` | `trips_full.csv` をODペア別に集計し、LineString GeoJSON／GeoParquetを出力 |
| `make_zone_polygons.py` | e-Stat 令和2年国勢調査 小地域データからゾーンポリゴンを生成。ゾーンコード表の大字町丁目名で名前マッチング（括弧除去・異体字変換・旧市町村名プレフィックス除去・大字サブ区画union・X町→X新開等の派生サブ区画取得・1文字ゾーン名の oaza_code 別共通プレフィックス確認など多段階正規化）→ e-Statにない地名は登記所備付地図（法務省）GeoJSONで補完 → `zone_polygons.geojson` / `zone_polygons.parquet` / `zone_coords.csv` / `zone_coords.geojson`（ポリゴン重心）を出力 |
| `geocode_zones.py` | 旧: 国土地理院APIでジオコーディング。現在は make_zone_polygons.py がポリゴン重心から zone_coords を生成するため不要 |

スクリプト内のパス: `ROOT_DIR`=プロジェクトルート、`DATA_DIR`=`data/`

### zone_polygons の属性

| フィールド | 内容 |
|---|---|
| `zone_key` | city_code(5桁) + town_code(3桁ゼロ埋め) の8桁キー |
| `city_code` / `town_code` | 市区町村コード・大字町コード |
| `city_name` / `town_name` | 市区町村名・大字町名 |
| `polygon_source` | `name`=e-Stat 町丁名マッチ（2206件）、`moj`=登記所備付地図大字名マッチ（2件） |

※ 3件（津山市加茂町齋野谷、高梁市川上町吉木、新庄村長床）は e-Stat・MOJ どちらにも存在しないためポリゴンなし（出力に含まれない）。
